## Estado y siguientes pasos

- **Pendiente de Arquitecto:** solución i18n y estructura de claves, decisión PWA/offline, formato de datos de contenido (JSON), estrategia de imprimible (CSS print recomendado), esquema de `localStorage`.
- **Pendiente de UX-UI:** universo visual festivo, avatares, voz/tono de textos, sistema de recompensas visible.
- **Pendiente de Abogado (gate):** validar ausencia de tratamiento RGPD, uso funcional de localStorage sin banner, originalidad del contenido frente a copyright editorial.
- **Pendiente de Responsabilidad Social (gate):** mecánicas de racha sanas (no dark patterns con menores), sostenibilidad de la ficha impresa.
- **Producción de contenido:** redactar ejercicios originales por tema (no incluido en estas specs).
